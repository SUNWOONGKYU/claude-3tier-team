# Claude 3계층 작업팀 구성법 (claude-3tier-team)

> 표시명: **Claude 3계층 작업팀 구성법** (2026-07-11 지휘관 지시로 개명. 폴더·슬러그는 등록 호환을 위해 claude-3tier-team 유지)

소대장–분대장–분대원 3계층 편제로 임무를 분해·병렬 수행하고, 이중 검증 게이트를 통과한 산출물만 지휘관에게 보고하는 작업 편제 스킬.

## 1. 편제 (확정 — 변경 금지)

| 직책 | 실체 | 모델 | 과금 | 소환/호출 주체 |
|---|---|---|---|---|
| 지휘관 | 사용자 | — | — | — |
| 소대장 (1계층) | 팀 리더, 상주 (현재 세션) | Opus 4.8 | 구독 | 지휘관이 임무 하달 |
| 분대장 (2계층) | 팀메이트 | Sonnet 5 | 구독 | 소대장이 소환 + 모델 배정 |
| 분대원 (3계층) | 서브에이전트 | Haiku 4.5 기본, 필요시 Sonnet 5 | 구독 | 분대장이 소환 + 모델 배정 |
| 검증담당 V① | 팀메이트(읽기 전용) | Sonnet 5 | 구독 | 소대장 |
| 검증담당 V② | 팀메이트(읽기 전용) | Opus 4.8 | 구독 | 소대장 |
| 기획·전략참모 | API 호출 (상주 없음) | Fable 5 (claude-fable-5) | API 건당 | 소대장 판단 |
| 용병 | 외부 AI 5종 | ChatGPT, Gemini, Grok, Perplexity, GLM | 경로별 상이 | 소대장 판단 |

**작업 흐름**: ① 임무 → ② 소대장이 분대장 소환+모델 배정 → ③ 분대장이 필요시 분대원 소환+모델 배정 → ④ 산출물 제출 → ⑤ V①→V② 검증 통과 → 소대장 종합 → ⑥ 지휘관 보고 → 팀 정리

## 2. 편제 원칙 (8개 — 위반 금지)

1. **모델 배정은 소환자가 결정한다** — 소대장→분대장, 분대장→분대원. spawn 시 `model` 매개변수로 전달한다.
2. **모델은 소환 시점에 고정된다** — 사후 변경 불가. 재배정은 종료 후 재소환으로만 한다. 따라서 운용 순서는 반드시 "업무 파악 → 모델 결정 → 소환".
3. **팀메이트 중첩 소환 불가** — 분대원은 반드시 서브에이전트(Agent/Task 도구)로 편성한다. 팀메이트가 팀메이트를 만드는 구조는 금지.
4. **분대원은 컨텍스트 격리 상태로 시작** — 분대장은 소환 프롬프트에 완결적 임무명세서(대상, 범위, 기준, 보고 형식)를 담아야 한다.
5. **분대원은 자기 분대장에게만 보고하고 소멸한다.**
6. **검증자는 수정 권한이 없다** — tools를 Read, Grep, Glob으로 제한 (Edit·Write 금지). 발견·보고만 하고, 수정은 작업자(해당 분대장)가 하며, 수정본은 재검증한다 (순환 검증).
7. **검증은 2단 게이트** — V① 전수 1차(형식·기준·명백한 오류 필터) → 통과분만 V② 최종 게이트(판단 품질, 통과/반려 결정) 순차 진행.
8. **용병과 참모는 부대원이 아니라 호출 대상이다** — 상주시키지 않는다.

## 3. 모델 라우팅 기준표

소대장이 작업을 분해할 때 아래 기준으로 모델을 배정한다:

- **Opus 4.8** (`opus`): 판단·종합·아키텍처 결정·최종 검증 게이트. 틀린 답의 비용이 큰 자리에만.
- **Sonnet 5** (`sonnet`): 구현·코딩·문서·리뷰 등 주력 작업 전반. 분대장 기본 모델.
- **Haiku 4.5** (`haiku`): 대량 탐색·분류·요약·포맷 변환 등 물량전. 분대원 기본 모델.
- **Fable 5 (API)** (`claude-fable-5`): 기획·전략 자문이 필요할 때만 소대장이 호출, 자문 결과만 회수.
- **비용 규칙**: 분대장은 자기 작업이 병렬 분해 가능하거나 대량 탐색일 때만 분대원을 소환한다. 분대원 기본은 Haiku 4.5, 판단이 섞인 작업만 Sonnet 5.

## 4. 운용 절차 (소대장 행동 규칙)

### 4.1 임무 접수 → 팀 편성
1. **업무 파악 먼저**: 임무를 읽고 작업을 담당 구역 단위로 분해한다. (모델은 소환 시 고정되므로 파악이 소환보다 먼저다.)
2. **모델 결정**: §3 라우팅 기준표로 각 구역의 분대장 모델을 정한다 (기본 `sonnet`).
3. **분대장 소환**: 팀메이트로 소환한다 — "Spawn a teammate named squad-A using the claude-3tier-squad-leader agent type, model sonnet" 방식으로 역할 파일 + 이름 + 모델을 명시 배정한다 (역할 파일의 tools·model·본문이 팀메이트에 적용됨, §4.4 참조). 소환 프롬프트에 담당 구역·산출물·완료 기준을 명시. 검증담당 V①·V②도 같은 방식으로 각 역할 파일을 지정해 팀메이트로 소환한다.
4. **모델 배정 실반영 검증 (필수)**: 소환 직후 각 분대장·분대원의 실제 모델을 소환 로그/응답으로 확인한다. 과거(2026-03) `model` 매개변수가 무시되는 버그가 보고된 바 있다. 명시 배정이 /config 기본값보다 항상 우선해야 하며, 불일치 시 종료 후 재소환한다.

### 4.2 검증 게이트
1. 분대장 산출물이 모이면 V①(`claude-3tier-verifier-v1`, model: sonnet)을 소환해 **전수 1차 검증** (형식·기준 충족·명백한 오류).
2. V① 통과분만 V②(`claude-3tier-verifier-v2`, model: opus)에 넘겨 **최종 게이트** (판단 품질, 통과/반려 결정).
3. 반려분은 해당 분대장이 수정 → V①부터 재검증 (순환 검증). 검증자가 직접 수정하는 것은 금지.

### 4.3 종합 · 보고 · 해산
1. V② 통과분을 소대장이 종합해 지휘관에게 보고한다.
2. 보고에는 편성표(누가·어떤 모델·어떤 구역), 검증 결과(통과/반려/재검증 횟수), 산출물 목록을 포함한다.
3. 보고 후 팀을 정리(해산)한다.

### 4.4 팀메이트(Agent Teams) 공식 규격 — 소환 전 숙지 (출처: code.claude.com/docs/en/agent-teams)

**활성화·환경**
- 실험 기능: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 필요 (settings.json `env` 또는 셸 환경변수). v2.1.178 이상에서 지원 (v2.1.201에서 검증됨).
- 팀은 첫 팀메이트 소환 시 자동 생성된다 (v2.1.178+). 팀 설정: `~/.claude/teams/{팀명}/config.json`, 공유 작업목록: `~/.claude/tasks/{팀명}/`.
- 세션당 팀은 정확히 1개. 리더 교체 불가. `/resume`·`/rewind` 후 in-process 팀메이트는 복원되지 않는다 — 세션이 끊기면 재소환한다.
- Windows Terminal은 split pane 미지원 — Windows에서는 in-process 모드(기본)로 운용한다. 조작: ↑/↓ 팀메이트 선택, Enter 대화 열기, x 중지, Esc 복귀.

**소환·역할 파일**
- 분대장·검증담당은 서브에이전트 정의를 팀메이트에 재사용한다: "Spawn a teammate using the claude-3tier-squad-leader agent type ..." 방식. 이때 역할 파일의 `tools` 제한·`model`·본문(추가 시스템 프롬프트)이 팀메이트에 적용된다. 단 `skills`·`mcpServers` 필드는 적용되지 않는다 (프로젝트/사용자 설정에서 로드됨).
- 팀메이트 소환은 리더(소대장)만 가능 — 팀메이트가 팀메이트를 소환할 수 없다 (편제 원칙 3과 일치). 분대장(팀메이트)의 분대원은 서브에이전트로 소환하되, **in-process 팀메이트는 백그라운드 서브에이전트를 못 쓴다 — 분대원은 포그라운드로 소환**시킨다.

**모델 배정 (공식 확인)**
- 팀메이트는 리더의 /model을 상속하지 않는다. 우선순위: 소환 프롬프트의 명시 배정 > /config "Default teammate model". 이 스킬은 항상 명시 배정한다.
- 팀메이트의 모델·fast mode는 소환 시점에 고정 — 사후 변경 불가 (편제 원칙 2와 일치). 팀메이트 세션의 /model·/fast는 리더에만 영향.
- 팀메이트는 리더의 effort level과 권한 설정을 소환 시점에 상속한다.

**교신·해산**
- 팀메이트 간 직접 교신: SendMessage (이름으로 지정). 공유 Task List로 자율 조정. 유휴 시 리더에 자동 알림 — 폴링 불필요.
- 해산: 리더가 종료 요청 → 팀메이트가 승인/거부 (진행 중 API 요청 완료 후 종료라 지연 가능). 팀 config는 세션 종료 시 자동 삭제, task list는 `cleanupPeriodDays`(기본 30일)로 정리.

**알려진 이슈 (운용 시 주의)**
- 팀메이트가 Task 완료 표시를 놓치면 의존 Task가 막힌다 — 소대장은 공유 작업목록 상태를 주기 점검한다.
- 팀메이트가 API 오류로 멈추면 자동 복구되지 않는다 — 소대장이 재지시하거나 재소환한다.
- 권한 승인 프롬프트는 리더 화면에서만 뜬다 — 팀메이트 간 권한 요청 릴레이 불가.
- 토큰 비용은 단일 세션보다 훨씬 높다 (팀메이트별 독립 컨텍스트) — 병렬 이득이 있을 때만 팀을 꾸린다. 순차 작업이면 서브에이전트가 낫다.

## 5. 기획·전략참모 (Fable 5) 호출

- 팀메이트로 소환하지 않는다 (구독 세션과 API 인증 혼합 불가). 상주 없음, 건당 과금.
- **설계 전제**: 2026-07-12 이후 Fable 5는 구독 미포함(API 전용)이다. 따라서 참모는 언제나 API 호출로만 접근하며, 구독 경로 폴백은 두지 않는다.
- 소대장이 기획·전략 판단이 필요할 때만 아래 스크립트로 호출하고 응답만 회수한다:

```
python ~/.claude/skills/claude-3tier-team/scripts/call-fable5.py "자문 요청 프롬프트"
# 또는 파일 입력: python .../call-fable5.py --file 질문.md
```

- API 키는 환경변수 `ANTHROPIC_API_KEY`로 관리한다. 스크립트에 하드코딩 금지.
- 키 미설정 시 스크립트가 안내 메시지를 내고 종료한다 — 이때 소대장은 지휘관에게 키 설정을 요청하거나 참모 없이 진행한다.

## 6. 용병 호출 규칙 (외부 AI 5종)

소대장 판단으로만 호출. 호출 경로 우선순위 (폴백):

**① CLI 우선 → ② 웹세션 자동화 (크롬·구독, 추가 비용 없음) → ③ API/MCP (건당 과금)**

| 용병 | ① CLI | ② 웹세션 자동화 | ③ API/MCP |
|---|---|---|---|
| ChatGPT | `codex` 명령 (Codex CLI, ChatGPT 계정 로그인) | chatgpt.com | OpenAI API |
| Gemini | `gemini` (Gemini CLI) | gemini.google.com | Gemini API |
| Grok | Grok CLI (실시간 X 데이터 접근 가능) | grok.com | xAI API |
| Perplexity | 전용 CLI 확인 필요 — 없으면 ②로 | perplexity.ai | Perplexity API / MCP |
| GLM | 전용 CLI 없음 — 기존 코딩 CLI에 GLM Coding Plan 연동 또는 ZCode | chat.z.ai | Z.ai API (OpenAI·Anthropic 프로토콜 호환) |

폴백 로직:
1. 해당 CLI가 설치·로그인돼 있으면 CLI로 호출한다 (`Get-Command` 로 존재 확인).
2. CLI 불가 시 기존 `웹세션-자동화` 스킬(크롬, Playwright+CDP, chat 모드)을 호출해 해당 사이트에 질의한다. 신규 개발이 아니라 호출 연동이다.
3. 웹세션도 불가할 때만 API/MCP(건당 과금)로 호출하되, 과금 발생을 지휘관 보고에 명시한다.

표기 규칙: 스킬 문서와 사용자 대면 출력에서는 "ChatGPT"로 표기한다. Codex라는 이름은 CLI 명령 매핑에만 쓴다.
